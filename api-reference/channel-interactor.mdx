`ChannelInteractor` is the primary interface for all channel-level operations in the Sceyt Chat Android UIKit. It is accessible via `SceytChatUIKit.chatUIFacade.channelInteractor`. All methods are `suspend` functions unless stated otherwise, and most return either a `SceytResponse<T>` (for single-result operations) or a `Flow<PaginationResponse<SceytChannel>>` (for paginated list operations). `SceytResponse` is a sealed class with `Success` and `Error` variants, making exhaustive handling straightforward.

**Package:** `com.sceyt.chatuikit.persistence.interactor`

```kotlin
val interactor: ChannelInteractor = SceytChatUIKit.chatUIFacade.channelInteractor
```

## Loading Channels

<ResponseField name="loadChannels" type="suspend fun → Flow&lt;PaginationResponse&lt;SceytChannel&gt;&gt;">
  Loads a paginated list of channels. Results are emitted from the local database first, then updated with the server response. The flow closes when all data has been delivered.

  ```kotlin
  interactor.loadChannels(
      offset           = 0,
      searchQuery      = "",
      loadKey          = LoadKeyData(),
      onlyMine         = false,
      ignoreDb         = false,
      awaitForConnection = false,
      config           = ChannelListConfig.default
  ).collect { response ->
      when (response) {
          is PaginationResponse.DBResponse -> updateUiFromDb(response.data)
          is PaginationResponse.ServerResponse -> updateUiFromServer(response.data)
          else -> {}
      }
  }
  ```

  <Expandable title="Parameters">
    <ParamField path="offset" type="Int" required>
      Zero-based pagination offset (number of items already loaded).
    </ParamField>
    <ParamField path="searchQuery" type="String" required>
      Text to filter channel subjects. Pass an empty string to load all channels.
    </ParamField>
    <ParamField path="loadKey" type="LoadKeyData?" required>
      A key used to deduplicate in-flight requests and correlate responses; pass `null` or `LoadKeyData()` for a fresh load.
    </ParamField>
    <ParamField path="onlyMine" type="Boolean" required>
      When `true`, only channels where the current user is a member are returned.
    </ParamField>
    <ParamField path="ignoreDb" type="Boolean" required>
      When `true`, skips the local database and fetches directly from the server.
    </ParamField>
    <ParamField path="awaitForConnection" type="Boolean" required>
      When `true`, suspends the call until the client is connected before issuing a server request.
    </ParamField>
    <ParamField path="config" type="ChannelListConfig" required>
      Specifies channel types, sort order, page size, and query parameters. Use `ChannelListConfig.default` for standard behaviour.
    </ParamField>
  </Expandable>
</ResponseField>

<ResponseField name="searchChannelsWithUserIds" type="suspend fun → Flow&lt;PaginationResponse&lt;SceytChannel&gt;&gt;">
  Searches channels that contain the specified user IDs, optionally also matching a subject query. Useful for finding direct or group channels with specific participants.

  ```kotlin
  interactor.searchChannelsWithUserIds(
      offset                       = 0,
      searchQuery                  = "",
      userIds                      = listOf("user-abc", "user-xyz"),
      config                       = ChannelListConfig.default,
      includeSearchByUserDisplayName = true,
      onlyMine                     = true,
      ignoreDb                     = false,
      loadKey                      = null,
      directChatType               = "direct"
  ).collect { response -> /* handle */ }
  ```

  <Expandable title="Parameters">
    <ParamField path="offset" type="Int" required>
      Zero-based pagination offset.
    </ParamField>
    <ParamField path="searchQuery" type="String" required>
      Optional subject text filter. Pass an empty string to match any subject.
    </ParamField>
    <ParamField path="userIds" type="List&lt;String&gt;" required>
      IDs of users that must be members of the returned channels.
    </ParamField>
    <ParamField path="config" type="ChannelListConfig" required>
      Channel list configuration (types, order, limit, query params).
    </ParamField>
    <ParamField path="includeSearchByUserDisplayName" type="Boolean" required>
      When `true`, also matches channels whose members' display names contain the `searchQuery`.
    </ParamField>
    <ParamField path="onlyMine" type="Boolean" required>
      When `true`, restricts results to channels the current user belongs to.
    </ParamField>
    <ParamField path="ignoreDb" type="Boolean" required>
      When `true`, bypasses the local database cache.
    </ParamField>
    <ParamField path="loadKey" type="LoadKeyData?" required>
      Deduplication key for the request.
    </ParamField>
    <ParamField path="directChatType" type="String">
      The channel-type identifier for direct chats. Default: `"direct"`.
    </ParamField>
  </Expandable>
</ResponseField>

<ResponseField name="getChannelsBySQLiteQuery" type="suspend fun → List&lt;SceytChannel&gt;">
  Executes a raw SQLite query against the local channel database. Use for advanced offline filtering not supported by the standard API.

  ```kotlin
  val query = SimpleSQLiteQuery("SELECT * FROM channels WHERE pinned = 1")
  val pinnedChannels: List<SceytChannel> = interactor.getChannelsBySQLiteQuery(query)
  ```

  <Expandable title="Parameters">
    <ParamField path="query" type="SimpleSQLiteQuery" required>
      A `SimpleSQLiteQuery` wrapping a raw SQL string with optional bind arguments.
    </ParamField>
  </Expandable>
</ResponseField>

<ResponseField name="syncChannels" type="suspend fun → Flow&lt;SyncResult&lt;SceytChannel&gt;&gt;">
  Forces a full server-side channel sync using the supplied config. Emits `SyncResult` events as pages of channels are fetched and persisted.

  ```kotlin
  interactor.syncChannels(ChannelListConfig.default)
      .collect { result -> /* handle sync progress */ }
  ```

  <Expandable title="Parameters">
    <ParamField path="config" type="ChannelListConfig" required>
      Configuration governing which channel types and how many to sync per page.
    </ParamField>
  </Expandable>
</ResponseField>

</ResponseField>

## Unread Counts and Draft

<ResponseField name="getTotalUnreadCount" type="fun → Flow&lt;Long&gt;">
  Returns a hot `Flow` that emits the total unread message count across channels, optionally filtered by channel type. Emits a new value whenever the count changes.

  ```kotlin
  interactor.getTotalUnreadCount(channelTypes = listOf("direct", "group"))
      .onEach { count -> updateBadge(count) }
      .launchIn(lifecycleScope)
  ```

  <Expandable title="Parameters">
    <ParamField path="channelTypes" type="List&lt;String&gt;">
      Channel type identifiers to include in the count. Pass an empty list (default) to count all channel types.
    </ParamField>
  </Expandable>
</ResponseField>

<ResponseField name="getChannelMessageCount" type="fun → Flow&lt;Long&gt;">
  Returns a hot `Flow` that emits the total number of messages in a specific channel from the local database.

  <Expandable title="Parameters">
    <ParamField path="channelId" type="Long" required>
      The numeric ID of the channel.
    </ParamField>
  </Expandable>
</ResponseField>

<ResponseField name="updateDraftMessage" type="suspend fun">
  Persists a draft message for the channel to the local database. The draft is displayed in the channel list and restored when the user reopens the conversation.

  ```kotlin
  interactor.updateDraftMessage(
      DraftMessage(channelId = 42L, message = "I was typing this...")
  )
  ```

  <Expandable title="Parameters">
    <ParamField path="draftMessage" type="DraftMessage" required>
      A `DraftMessage` object containing the channel ID and the draft text (and optional reply/mention data).
    </ParamField>
  </Expandable>
</ResponseField>
