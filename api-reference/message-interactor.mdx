`MessageInteractor` is the primary interface for all message-level operations in the Sceyt Chat Android UIKit. It is accessible via `SceytChatUIKit.chatUIFacade.messageInteractor`. Nearly all methods are `suspend` functions. Paginated loading methods return `Flow<PaginationResponse<SceytMessage>>`, which emits a database result first and then updates from the server. One-shot operations return `SceytResponse<T>` — a sealed class with `Success` and `Error` variants. Thread replies are supported throughout by passing `replyInThread = true` and using the parent message ID as the `conversationId`.

**Package:** `com.sceyt.chatuikit.persistence.interactor`

```kotlin
val interactor: MessageInteractor = SceytChatUIKit.chatUIFacade.messageInteractor
```

## Loading Messages

<ResponseField name="loadPrevMessages" type="suspend fun → Flow&lt;PaginationResponse&lt;SceytMessage&gt;&gt;">
  Loads messages older than `lastMessageId` (backwards pagination). Emits cached messages immediately, then emits the server response once available.

  ```kotlin
  interactor.loadPrevMessages(
      conversationId = channelId,
      lastMessageId  = oldestVisibleMessageId,
      replyInThread  = false,
      offset         = 0,
      limit          = 50,
      loadKey        = LoadKeyData(),
      ignoreDb       = false
  ).collect { response -> /* render */ }
  ```

  <Expandable title="Parameters">
    <ParamField path="conversationId" type="Long" required>
      Channel ID, or the parent message ID when `replyInThread` is `true`.
    </ParamField>
    <ParamField path="lastMessageId" type="Long" required>
      The ID of the oldest message currently loaded. Older messages are fetched relative to this.
    </ParamField>
    <ParamField path="replyInThread" type="Boolean" required>
      Pass `true` to load thread replies; `false` for the main conversation.
    </ParamField>
    <ParamField path="offset" type="Int" required>
      Zero-based count of messages already loaded in this direction.
    </ParamField>
    <ParamField path="limit" type="Int">
      Maximum number of messages to fetch. Defaults to `config.queryLimits.messageListQueryLimit`.
    </ParamField>
    <ParamField path="loadKey" type="LoadKeyData" required>
      Deduplication and correlation key for the request.
    </ParamField>
    <ParamField path="ignoreDb" type="Boolean">
      When `true`, bypasses the local database. Default: `false`.
    </ParamField>
  </Expandable>
</ResponseField>

<ResponseField name="loadNextMessages" type="suspend fun → Flow&lt;PaginationResponse&lt;SceytMessage&gt;&gt;">

    </ParamField>
    <ParamField path="lastMessageId" type="Long" required>
      The ID of the newest message currently loaded.
    </ParamField>
    <ParamField path="replyInThread" type="Boolean" required>
      Pass `true` for thread replies.
    </ParamField>
    <ParamField path="offset" type="Int" required>
      Zero-based count of messages already loaded in this direction.
    </ParamField>
    <ParamField path="limit" type="Int">

    <ParamField path="ignoreDb" type="Boolean">

<ResponseField name="loadNearMessages" type="suspend fun → Flow&lt;PaginationResponse&lt;SceytMessage&gt;&gt;">
  Loads messages surrounding a specific `messageId` — useful for scrolling to a search result or a mention deep inside the history. Fetches both older and newer messages around the anchor.

  ```kotlin
  interactor.loadNearMessages(
      conversationId = channelId,
      messageId      = targetMessageId,
      replyInThread  = false,
      limit          = 50,
      loadKey        = LoadKeyData(),
      ignoreDb       = false,
      ignoreServer   = false
  ).collect { response -> /* render */ }
  ```

  <Expandable title="Parameters">
    <ParamField path="conversationId" type="Long" required>
      Channel ID, or thread parent message ID.
    </ParamField>
    <ParamField path="messageId" type="Long" required>
      The ID of the anchor message around which to load context.
    </ParamField>
    <ParamField path="replyInThread" type="Boolean" required>
      Pass `true` for thread replies.
    </ParamField>
    <ParamField path="limit" type="Int" required>
      Number of messages to load on each side of the anchor.
    </ParamField>
    <ParamField path="loadKey" type="LoadKeyData" required>
      Deduplication key for the request.
    </ParamField>
    <ParamField path="ignoreDb" type="Boolean">
      When `true`, skips the local database. Default: `false`.
    </ParamField>
    <ParamField path="ignoreServer" type="Boolean">
      When `true`, only returns local data without a server call. Default: `false`.
    </ParamField>
  </Expandable>
</ResponseField>

## Searching Messages

<ResponseField name="searchMessages" type="suspend fun → SceytPagingResponse&lt;List&lt;SceytMessage&gt;&gt;">
  Performs a full-text search in a conversation. Returns the first page of matching messages. Call `loadNextSearchMessages()` to paginate through additional results.

  ```kotlin
  val result = interactor.searchMessages(
      conversationId = channelId,
      replyInThread  = false,
      query          = "hello world"
  )
  ```

  <Expandable title="Parameters">
    <ParamField path="conversationId" type="Long" required>
      The channel ID to search within.
    </ParamField>
    <ParamField path="replyInThread" type="Boolean" required>
      Pass `true` to search within a thread.
    </ParamField>
    <ParamField path="query" type="String" required>
      The search text to match against message bodies.
    </ParamField>
  </Expandable>
</ResponseField>

<ResponseField name="loadNextSearchMessages" type="suspend fun → SceytPagingResponse&lt;List&lt;SceytMessage&gt;&gt;">
  Loads the next page of results from the most recent `searchMessages` call. Must be called after `searchMessages` has returned successfully.

  ```kotlin
  val nextPage = interactor.loadNextSearchMessages()
  ```
</ResponseField>

## Unread Mentions

<ResponseField name="getUnreadMentions" type="suspend fun → SceytPagingResponse&lt;List&lt;Long&gt;&gt;">
  Returns a page of message IDs where the current user has been mentioned and has not yet read, relative to `messageId` in the specified `direction`.

  ```kotlin
  val mentions = interactor.getUnreadMentions(
      conversationId = channelId,
      direction      = Direction.Prev,
      messageId      = latestReadMessageId,
      limit          = 30
  )
  ```

  <Expandable title="Parameters">
    <ParamField path="conversationId" type="Long" required>
      The channel ID to query.
    </ParamField>
    <ParamField path="direction" type="Direction" required>
      `Direction.Prev` for older mentions, `Direction.Next` for newer.
    </ParamField>
    <ParamField path="messageId" type="Long" required>
      Anchor message ID for directional pagination.
    </ParamField>
    <ParamField path="limit" type="Int">
      Number of mention IDs to return per page. Defaults to `config.queryLimits.unreadMentionsListQueryLimit`.
    </ParamField>
  </Expandable>
</ResponseField>

## Sending Messages

<ResponseField name="sendMessageAsFlow" type="suspend fun → Flow&lt;SendMessageResult&gt;">
  Sends a message and returns a `Flow` that emits `SendMessageResult` events tracking the send lifecycle (pending → uploading attachments → sent). Prefer this over `sendMessage` when you need progress updates.

  ```kotlin
  interactor.sendMessageAsFlow(channelId, message)
      .onEach { result -> /* update UI for pending/sent states */ }
      .launchIn(lifecycleScope)
  ```

  <Expandable title="Parameters">
    <ParamField path="channelId" type="Long" required>
      The channel to send the message to.
    </ParamField>
    <ParamField path="message" type="Message" required>
      The `com.sceyt.chat.models.message.Message` object to send, including body, attachments, and metadata.
    </ParamField>
  </Expandable>
</ResponseField>

<ResponseField name="sendMessage" type="suspend fun">
  Fires a message send without returning a flow result. Use for fire-and-forget sending when you don't need per-message progress feedback.

  <Expandable title="Parameters">

    </ParamField>
    <ParamField path="message" type="Message" required>
      The message object to send.
    </ParamField>
  </Expandable>
</ResponseField>

<ResponseField name="sendMessages" type="suspend fun">

## Editing and Deleting

<ResponseField name="editMessage" type="suspend fun → SceytResponse&lt;SceytMessage&gt;">
  Edits an existing message's body or metadata. Subject to the `messageEditTimeout` window configured in `SceytChatUIKitConfig`.

  ```kotlin
  val result = interactor.editMessage(channelId, updatedMessage)
  ```

  <Expandable title="Parameters">
    <ParamField path="channelId" type="Long" required>
      The channel the message belongs to.
    </ParamField>
    <ParamField path="message" type="SceytMessage" required>
      The `SceytMessage` with updated `body` and/or `metadata` fields.
    </ParamField>
  </Expandable>
</ResponseField>

<ResponseField name="deleteMessage" type="suspend fun → SceytResponse&lt;SceytMessage&gt;">
  Deletes a message. The `deleteType` parameter controls whether the message is deleted only for the current user or for all participants.

  ```kotlin
  interactor.deleteMessage(
      channelId  = channelId,
      message    = messageToDelete,
      deleteType = DeleteMessageType.DeleteForEveryone
  )
  ```

  <Expandable title="Parameters">
    <ParamField path="channelId" type="Long" required>
      The channel the message belongs to.
    </ParamField>
    <ParamField path="message" type="SceytMessage" required>
      The message object to delete.
    </ParamField>
    <ParamField path="deleteType" type="DeleteMessageType" required>
      `DeleteMessageType.DeleteForMe` or `DeleteMessageType.DeleteForEveryone`. The `hardDeleteMessageForAll` config flag may override this.
    </ParamField>
  </Expandable>
</ResponseField>

## Channel Events and Live Flow

  ```kotlin

  ```

  <Expandable title="Parameters">
    <ParamField path="channelId" type="Long" required>
      The target channel.
    </ParamField>
    <ParamField path="event" type="String" required>
      The event type string (e.g. `"typing"`, `"recording"`).
    </ParamField>
  </Expandable>
</ResponseField>

<ResponseField name="getOnMessageFlow" type="fun → SharedFlow&lt;Pair&lt;SceytChannel, SceytMessage&gt;&gt;">
  Returns a hot `SharedFlow` that emits a `(SceytChannel, SceytMessage)` pair whenever a new message arrives on any channel. Subscribe at the application level to drive notifications or badge updates.

  ```kotlin
  interactor.getOnMessageFlow()
      .onEach { (channel, message) -> showNotification(channel, message) }
      .launchIn(lifecycleScope)
  ```
</ResponseField>
